# オーバーレイ実装

## Content Script の構造

- **Shadow DOM**: スタイルの競合を防ぐため、すべてのオーバーレイ要素を closed shadow root 内に配置
- **スポットライト**: `box-shadow: 0 0 0 9999px rgba(0,0,0,0.5)` でページ全体を暗くするマスクを作成し、対象要素を「穴」として表示
- **ツールチップ**: ヒントテキスト、ステップの進捗状況、戻る/次へ/停止ボタンを含むフローティング指示バブル
- **要素ファインダー**: `waitForElement()` は MutationObserver を使用して、対象要素が表示されるまで最大 5 秒間待機（動的なページに対応）

## ツールチップの配置

右 → 左 → 上 → 下の順に試行し、ビューポートに収まる最初の配置を選択します。対象要素に対して垂直/水平方向に中央揃えされます。

## 要素マッチング

再生時、`targetSelector` は文字列または文字列の配列のいずれかです。要素ファインダーは各セレクターを順番に試行し、最初に一致したものを返します。これにより DOM の状態差異に対応できます（例：コンボボックスの開閉前後）。

対象要素がすぐに利用できない場合（動的なページ、ナビゲーション後）、`waitForElement()` は MutationObserver を使用して DOM の変更を監視し、要素が表示されるか 5 秒のタイムアウトに達するまでマッチングをリトライします。
